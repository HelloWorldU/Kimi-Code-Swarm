# Debug Skill

> Agent 遇到代码层面的 bug 时遵循此规范。禁止盲猜。

---

## 核心原则

**看不出根因时，加日志而不是盲猜。**

---

## 标准步骤

1. **定位** — 通过日志（`src/utils/logger.ts`）把运行时状态打出来
2. **修复** — 基于数据而非猜测做修改
3. **留痕** — 修复后在代码注释或 commit message 中记录根因
4. **验证** — 改完后必须 build/test/lint/analyze 全部通过

---

## 日志规范

- 关键路径（外部调用/状态变更）必须有 `log.error` / `log.info`
- 非关键路径允许用 `// expected: ...` 意图注释替代日志
- 敷衍注释（`// ignore` / `// TODO`）不算

---

## 反模式

- ❌ 同一 bug 反复出现，每次都靠猜修复
- ❌ 增加规则/配置来掩盖根因，而不是定位根因
- ❌ 修复后不在注释或 commit 中记录根因
