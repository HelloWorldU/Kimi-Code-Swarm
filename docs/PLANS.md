## 定期检查机制（仓库一致性检查器）

> 一键启动的自动化扫描任务，从有限信息入口（AGENTS.md / docs/ / harness/）读取预期结构，检查仓库实际状态是否一致。

### 目标

Agent 运行 `npm run health-check`（或类似命令）即可触发扫描，发现以下问题：

| 检查项 | 说明 | 信息来源 |
|--------|------|---------|
| **文档引用完整性** | docs/ 中引用的文件路径是否实际存在 | `docs/**/*.md` 中的链接 |
| **代码文件冗余** | 是否有未被文档/AGENTS.md 提及的孤立文件 | `AGENTS.md` 目录结构 + `docs/doc-map.json` |
| **Harness 模板漂移** | harness/*.yaml 中的步骤是否与实际执行一致 | `harness/*.yaml` vs 实际代码 |
| **规则失效** | ast/rules/ 中的规则是否覆盖了当前代码库 | AST 规则 vs 实际源码 |
| **死代码检测** | 导出但未使用的函数/组件 | 静态分析 |
| **文档过时** | AGENTS.md 中的目录结构描述是否与实际一致 | `AGENTS.md` vs `tree` |

### 设计原则

- **只读检查**：不自动修改任何文件，只输出问题列表
- **有限入口**：只从 AGENTS.md、docs/、harness/ 等明确的信息源读取，不猜测
- **可配置忽略**：允许标记预期内的例外（如 `evals/` 当前为空是已知状态）

### 现状

框架待设计。预期集成到 `scripts/` 或独立 `health/` 目录，后续由 `scripts/cleanup.ts` 调度执行。
