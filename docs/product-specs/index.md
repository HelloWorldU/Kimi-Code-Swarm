# Product Specs

> 本地 Agent 指挥中心 App 规格。

---

## 产品定位

**本地 Agent 指挥中心** —— 指挥官（用户）通过桌面 App 派任务、监进度、审结果。

不是 Dashboard，是**指挥部**。点击操作，不用记命令。

---

## 核心角色

| 角色 | 实体 | 职责 |
|------|------|------|
| **指挥官** | 用户 | 创建任务、下达指令、审阅 PR、合并代码 |
| **工人** | Kimi Code CLI 进程 | 在独立目录里执行具体开发任务 |
| **工位** | 本地目录（git clone/worktree） | 隔离工作空间，互不干扰 |

---

## 标准工作流

```
1. 指挥官打开 App
2. 点击"新建任务"
   → App 自动新建本地目录
   → git clone 目标仓库
   → 切出新分支
   → 启动 Kimi CLI 进程
3. 指挥官输入任务指令
   （如："实现用户登录模块，包含前后端"）
4. Agent 在独立目录里干活
   （写代码、测试、git commit）
5. Agent 完成 → 自动 git add/commit/push → 自动创建 PR（任何步骤失败时，Engine 将完整执行日志全量回传给 Agent 自主判断并修复，最多 3 轮）
6. 指挥官在 App 里审阅 PR diff
   → 点击"合并到 main" 或 "打回修改"
7. 任务完成，Agent 可释放或接新任务
```

---

## 关键约束

| 约束 | 说明 |
|------|------|
| **单账号** | 所有 Agent 共享一个 Kimi 账号的 Token 配额 |
| **本地优先** | 纯目录隔离，不依赖虚拟机/容器/云服务 |
| **App 体验** | 点击操作，不需要记忆命令行 |
| **PR 驱动** | 所有变更通过 PR 合并到 main，不直接推 |

## App 核心界面

### 任务列表（首页）

| 字段 | 说明 |
|------|------|
| 任务名称 | 指挥官自定义，如"登录模块重构" |
| 工人编号 | Agent-01, Agent-02... |
| 工作目录 | 本地路径，如 `~/workspace/agent-01` |
| Git 分支 | 当前开发分支名 |
| PR 状态 | 未提 / Open / 已合并 / 关闭 |
| 进度 | 未开始 / 进行中 / 待审阅 / 已完成 |
| Token 消耗 | 本次任务累计消耗 |
| 操作 | 查看详情 / 停止 / 删除 |

### 任务详情页

- **终端面板**：实时显示 Agent 的 CLI 输出（只读或交互）
- **Git 面板**：当前分支、commit 历史、diff 预览
- **PR 面板**：PR 标题、描述、文件变更列表、合并按钮
- **Token 面板**：消耗趋势、剩余配额估算

### 新建任务弹窗

1. 输入任务名称
2. 输入目标仓库 URL
3. 输入任务指令（给 Agent 的 Prompt）
4. 选择 Token 预算上限（可选）
5. 点击"启动" → App 全自动完成后续

## 状态机

```
[待启动] → [克隆中] → [就绪] → [工作中] → [待审阅] → [已完成]
                                      ↓
                                  [已停止]
```

| 状态 | 说明 |
|------|------|
| 待启动 | 用户已创建任务，尚未点击启动 |
| 克隆中 | 正在 git clone / 切分支 |
| 就绪 | CLI 进程已启动，等待用户输入指令 |
| 工作中 | Agent 正在执行指令 |
| 待审阅 | Agent 已完成并自动推送分支、创建 PR，等待全员审阅通过 |
| 已完成 | PR 已合并，任务结束 |

| 已停止 | 用户手动停止或进程异常退出 |

### 全员审阅门控

PR 创建后，所有**其他 Agent** 必须审阅通过，指挥官才能合并：
- 工作 Agent 提交 PR → 系统自动指派其余 N-1 个 Agent 审阅
- 每个审阅者可「通过」或「拒绝」
- 全员通过前，合并按钮禁用
- 任一拒绝则 PR 被打回（Agent 继续修改）

---

## 非目标（明确不做）

- ❌ 云端部署 / 远程 Agent
- ❌ 多账号支持（第一期只支持单 Kimi 账号）
- ⚡ 自动合并（全员审阅通过后 Engine 自动调用 GitHub API 合并；指挥官也可手动触发）
- ❌ 复杂编排（第一期不做 supervisor-worker 模式，每个 Agent 独立接任务）
